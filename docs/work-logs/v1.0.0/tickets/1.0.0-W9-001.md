## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### Problem Analysis：四個設計層根因

核心張力：`.claude/` 配置架構同時追求三目標，settings 雙層設計只滿足前兩個，犧牲第三個。

| 目標 | 滿足機制 |
|------|---------|
| 1. 可同步性（框架散佈/更新到下游） | settings.json 追蹤檔被 sync overlay |
| 2. 本地特化保護（私有配置不被覆蓋） | settings.local.json local-only，sync 不觸及 |
| 3. 參照完整性（註冊永遠指向存在的檔） | 無任何機制保證 ← 本問題根源 |

### 根因 1：註冊與檔案位置解耦，無參照完整性
hook 的「實際位置」（檔案系統）與「註冊」（JSON 字串路徑）是同一事實的兩份副本——即 tool-output-trust-rules 規則 5「記錄平面 vs 世界平面二相性」在配置層的翻版。搬檔不會自動更新指向它的字串註冊；一個 hook 可在 N 處註冊，無工具強制 N 份與 1 檔同步。file move silently breaks registration。

### 根因 2：同步邊界與職責邊界不對齊（盲區來源）
sync 為達目標 2 刻意排除 local 檔。但「hook 註冊」職責同時存在於兩層——一旦舊註冊殘留 local 層即永久豁免於上游修正。sync --audit 的孤兒稽核掃不到 local 檔內部的孤兒註冊。更深矛盾：sync 無法區分 local 內容是「刻意本地特化（該保護）」或「搬遷殘留孤兒（該清理）」，保護與修正在 local 層互斥。

### 根因 3：無「單一註冊來源」原則（最高槓桿修復點）
系統允許同一 hook 在兩檔各註冊一次但不協調（合併語意為疊加非去重）。無規則規定「hook 註冊一律歸 settings.json，local 只放 permissions/env/mcp」，故 local 層持續累積本該歸 canonical 的內容。本案的 handoff 幽靈與 agent-dispatch 重複皆源於此。

### 根因 4：驗證為「事後偵測」非「寫入時預防」
completeness-check 能報告 ghost/duplicate，但 WARNING 不阻擋、淹沒於 SessionStart、且只偵測已成幽靈者，不偵測 latent（目前合法、relocate 後必成幽靈）的 local hook 註冊。缺 relocate migration 工具與 pre-write path guard。

### 框架散佈架構的結構限制
CLAUDE.md §6.5「上游 blog → 散佈 → 下游 consumer」架構下，上游搬 hook 時改得了自己的 settings.json，但永遠碰不到下游各專案的 settings.local.json。「上游搬檔 + 下游 local 舊註冊」組合在此架構下結構上無法被上游自癒——只能靠下游各自發現各自修。這正是必須在設計層（規則 + 偵測）固化預防的理由。

### 觸發鏈

1. hook 原位於 `.claude/hooks/`，早期被註冊進 `settings.local.json`（當時正確）。
2. 框架將 hook 模組化搬遷至 `.claude/skills/ticket/hooks/`，並更新 canonical `settings.json`。
3. `settings.local.json` 為 sync 排除檔，舊註冊未被任何機制清理 → 形成幽靈指向。
4. 每次 Stop 事件 runtime 嘗試執行舊路徑 → `No such file or directory`。

### 連帶觀測

同一 `settings.local.json` 另有 `agent-dispatch-validation-hook.py` 與 settings.json 重複註冊（PreToolUse:Agent），additive 合併語意下重複執行。

### 已驗證的既有防護缺口

`hook-completeness-check.py`（W9-004 / framework issue #2）已偵測 ghost + duplicate 並列於 SessionStart，但：(a) WARNING 不阻擋；(b) 淹沒在十餘條 SessionStart warning 中被略過；(c) 僅偵測「已成幽靈/重複」者，對「目前合法但 relocate 後必成幽靈」的 local 層 hook 註冊無預防。
