## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### Context Bundle

**遷移背景**：Cloudflare Tunnel → Tailscale。憑證包從 v1（8 欄）縮為 v2（5 欄）。

**改動檔案**：`server/enroll.go`, `server/enroll_test.go`

**enroll.go 改動清單**：
1. `bundle` struct 改 v2（對齊 docs/contract.md）：V=2, Protocol, Endpoint, TtydUser, TtydPass（移除 CFAccessID/CFAccessSecret/ProxyToken）
2. `buildBundle` 只接收 endpoint, ttydUser, ttydPass
3. `runEnroll`：移除 cf-access-id/cf-access-secret/secret-backend/secret-file/keychain-service/keychain-account/reuse flag；移除 newToken() 和密鑰邏輯；移除 internal/secret import；endpoint 改 http://<tailscale-ip>:<port>/ws
4. 移除 `newToken` 函式、crypto/rand、encoding/hex import
5. 保留 `renderQR` 不變

**enroll_test.go 改動清單**：
1. 刪除 TestNewToken_Length64Hex、TestNewToken_Unique
2. 更新 TestBuildBundle：v2 格式（5 欄，V=2）
3. 新增 TestBuildBundle_V2Format（JSON 只有 5 key）、TestBuildBundle_Protocol

**殘留檢查**：無 CFAccessID/CFAccessSecret/ProxyToken/crypto/rand/encoding/hex

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

- `bundle` struct 從 8 欄縮為 5 欄（v2），V=2
- `buildBundle` 簡化為 3 參數（endpoint, ttydUser, ttydPass）
- `runEnroll` 移除 CF Access / secret-backend / reuse 等 flag 和密鑰邏輯
- 移除 `newToken()`、`crypto/rand`、`encoding/hex`、`internal/secret` import
- 測試更新：刪除 token 測試，新增 `TestBuildBundle_V2Format`、`TestBuildBundle_Protocol`
