#### 四、具體發現

##### [SEC-001] 中風險：WS 使用 ws:// 而非 wss://

- **位置**：`app/lib/features/terminal/protocol/ttyd_protocol.dart:61` — `buildUri()` 使用 `scheme: 'ws'`
- **影響**：Basic auth credentials 在 WS handshake 以 base64（可逆）傳輸。若 Tailscale 加密層失效（理論上不應發生），credentials 會以明文暴露
- **緩解因素**：Tailscale WireGuard 已提供端到端加密；本機 127.0.0.1 通訊也不經外部網路
- **建議**：在 proxy 加 TLS（自簽憑證即可），app 端改 `wss://`。或在文件中明確記錄「依賴 Tailscale 加密」的設計決策

##### [SEC-002] 中風險：keychain Store 密碼暫現 process args

- **位置**：`server/internal/secret/secret.go:72` — `exec.Command("security", "add-generic-password", ... "-w", value)`
- **影響**：密碼短暫出現在 `/proc/<pid>/cmdline`（macOS 的 `ps aux` 可見）
- **緩解因素**：程式碼註解已標記「自用本機可接受」；單人自用場景無其他用戶可觀察 process list
- **建議**：若需加固，改用 stdin pipe 傳密碼給 security CLI（`security` 支援 `-T` 或 stdin 模式）

##### [SEC-003] 低風險：enroll 憑證包 QR 碼無期限/無簽章

- **位置**：`server/enroll.go` — buildBundle() 不含 timestamp/expiry/signature
- **影響**：QR 圖片若被截圖，可永久使用
- **緩解因素**：enroll.go 提示「憑證包僅顯示這一次,勿截圖外流」；ttyd 密碼可透過 UC-03 輪替
- **建議**：已在設計決策 D7 明確取捨。無需修改

##### [SEC-004] 低風險：ConnectionManager 使用 developer.log 非統一日誌工具

- **位置**：`app/lib/features/terminal/connection/connection_manager.dart:169,201` — `developer.log()`
- **影響**：違反 observability-rules 規則 2（統一日誌工具）。debug build 日誌可能洩漏連線狀態資訊
- **建議**：替換為專案 AppLogger（若已建立）

##### [SEC-005] 低風險：deploy/launchd ttyd plist 含 placeholder 密碼

- **位置**：`deploy/launchd/com.apptunnel.ttyd.plist:16` — `user:CHANGE_ME`
- **影響**：使用者若未修改 placeholder 直接使用，會以弱密碼執行 ttyd
- **緩解因素**：plist 註解已提示替換；start.sh 要求 `-u` flag 不提供預設值
- **建議**：無需修改，placeholder 設計正確
