## Problem Analysis

### 統一：F4 + D = 同一「信念 vs 世界」斷層的事前/事後兩面

032+F-drill 收斂出單一元軸：決策失敗主體是「信念未對齊世界」。
- 事前面：F4 未驗假設就行動(39% of F) + F2 沒查既有就假設不存在(22%) = 行動前查證缺失，~60% of F。
- 事後面：D 信記錄非世界(24% 全語料) = confabulation/stale/假完成/漂移。
F4 與 D 不是兩問題，是「行動前不查」與「行動後信錯」。

### 兩軸的強制手段本質不同（關鍵設計約束）

- F（事前接地）：難機械化——要求行動者察覺「我在斷言沒查過的事」。只能靠**判斷協議**（WRAP Step0、最小實證、tool-output-trust 規則3）。框架裝最多 F 防護卻仍最大軸，正因協議是勸告型非強制型。
- D（事後接地）：可機械化——`git show` 等固定值、028/030 hook。

### 為何不按功能軸(A,19%)切

A 從沒卡過（路由順）；E(並行)僅 2.8%。把資源投在「信念接地」而非「分類/路由」，是資料指向。
