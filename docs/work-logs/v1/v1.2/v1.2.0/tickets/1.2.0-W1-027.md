## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

採「呼叫端注入已本地化字串」策略，將 OS 生物辨識提示文字沿認證鏈從持有
BuildContext 的 UI 層注入服務層：

1. `biometric_service.dart`：`authenticate()` 改為 `authenticate({required String localizedReason})`，
   service 契約改由呼叫端傳入提示文字，不在 service 內查 l10n。
2. `local_auth_biometric_service.dart`：使用注入的 `localizedReason`，移除硬編碼字串
   與 W1-019 deferred 例外註解。
3. `connection_manager.dart`：`connect({required String biometricReason})` /
   `reconnect({required String biometricReason})` 沿鏈傳遞至
   `_authenticateWithBiometrics(reason)`。
4. `terminal_screen.dart`：`_startConnection` / `_reconnect` 透過
   `AppLocalizations.of(context).authBiometricReason` 取得字串後注入。
   `_startConnection` 原於 initState 直接呼叫違反「inherited widget 查詢不可在
   initState 完成前執行」，改以 `addPostFrameCallback` 延後至首幀後啟動。

ARB key `authBiometricReason`（app/l10n/app_en.arb）由 W1-019 預留，本 ticket
接線至產生的 `AppLocalizations` getter。測試 fake 與
`connect()`/`reconnect()`/`authenticate()` 呼叫端同步更新以符合新簽章。
