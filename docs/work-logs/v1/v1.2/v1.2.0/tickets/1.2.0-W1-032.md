## 重現實驗結果

「重現」在此 ANA 等同「對全語料做可重跑的計數」。方法：(1) `ls` 七類目錄取全部檔名（檔名為描述性 slug，主要分類依據）；(2) 僅對語意模糊 slug 讀檔頭 title/根因判定；(3) 讀 `_project-registry.yaml` 取專案前綴對照。每條 PC/IMP/ARCH/... 歸一主軸，軸計數可由他人依同 slug 集重跑核對。

### (3) 主導軸判定

**主導軸為 F（知識/規格缺口，29.8%），D（狀態完整性，23.9%）緊隨為第二主軸。**

- 單一最大軸是 F，但 F 與 D 差距僅 5.9 個百分點（19 條），兩者共佔 53.7%，是語料的雙主導。
- 若以「process-compliance 子集」單看（194 條，最能代表 PM/agent 行為失誤）：F≈68、D≈58、B≈20、A≈18、C≈9、E≈9、G≈12 —— 同樣 F 領先、D 次之，雙主導格局在最相關子集上一致成立。
- A/G 合計 33.4% 屬「實作/品質類」雜訊軸（hook 誤判、程式壞味道），與「決策樹該卡哪軸」關聯較弱；扣除 A/G 後，決策相關軸（B/C/D/E/F）共 217 條，其中 F 佔 44.7%、D 佔 35.9%，雙主導更突出。

### (4) 與「1.2.0 並行樣本 D 主導」結論的對照（驗證 / 修正）

**結論：語料部分驗證、部分修正 1.2.0 並行樣本結論。**

| 面向 | 1.2.0 並行樣本 | 全語料實證 | 判定 |
|------|---------------|-----------|------|
| D 是否為主導軸之一 | 主導 | 第二主導（23.9%，決策子集 35.9%） | **驗證**——D 確實是真實高頻軸，非純並行偏誤 |
| D 是否「唯一」主導 | 暗示 D 獨大 | F 略高於 D（雙主導） | **修正**——並行樣本低估了 F（知識/規格缺口） |

- **驗證部分**：D 在全語料穩居第二（23.9%）、在決策相關子集達 35.9%，證明「狀態完整性（記錄≠世界）」是真實高頻失誤類別，1.2.0 觀察到 D 不是並行偶然。confabulation / 假完成 / 未提交 / 漂移 類問題確實密集。
- **修正部分**：1.2.0 單場樣本未充分暴露 F（知識/規格缺口）。F 才是全語料最大軸——「缺驗證就斷言、缺掃描就造輪、規格漏、未讀 code 直接判定」。單一 session 因任務同質，傾向反覆撞同類（D）失誤，系統性低估 F 這種「跨任務分散、單場密度低」的軸。這正是 WRAP Step0 警示的並行偏誤的具體形態：並行樣本放大了單場高頻軸（D），壓抑了長尾高總量軸（F）。
- **對決策樹體系化的啟示**：體系化不應只圍繞 D（狀態確認/ground-truth 查證）建護欄，必須對等投入 F（前置驗證/掃描/規格完整度閘門）。F+D 雙主導意味著決策樹的兩條主幹應是「動手前是否已查清楚（F）」與「相信的狀態是否等於世界真相（D）」。
