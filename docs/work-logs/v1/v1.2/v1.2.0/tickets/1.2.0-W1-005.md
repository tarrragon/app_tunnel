## Task Summary

process-skip-guard hook 的 keyword-pair 偵測對『引用/討論 skip-gate』的訊息誤觸（lexical false positive）。新增保守抑制：命中關鍵字落在引用 span（>/▎ 前綴行、echo 的 system-reminder 區）或與 meta 討論詞（假陽性/誤觸/固化/邊界/hook 提示）強共現時，視為討論非意圖，靜音。方向保守：寧可偶爾誤觸（PM 依 skip-gate.md 邊界 judge-past）也不可漏放真正跳過意圖（false negative 危險）

### 工作環境（重要）

實作於 **tarrragon/claude clone**：`/tmp/tcr-pr`，已在分支 `feat/process-skip-guard-reduce-false-trigger`（off origin/main，含已 merge 的 #5/#6）。
路徑無 `.claude/` 前綴——hook 在 `hooks/process-skip-guard-hook.py`，測試在 `hooks/tests/test_process_skip_guard_hook.py`。
**所有改動與 git/gh 都在 `/tmp/tcr-pr`，不要碰 app_tunnel 本地。** 完成 push 分支 + `gh pr create --repo tarrragon/claude`。

### 問題根因

`detect_skip_intent`（`hooks/process-skip-guard-hook.py` ~154）以 **keyword-pair** 偵測：`SKIP_SA_REVIEW` 在 `("跳過","sa")` / `("跳過","系統分析")` 同時出現即觸發。已有 `_should_silence_sa_review`（DOC/ANA + IMP Phase 4 靜音）。但對**引用/討論 skip-gate 的訊息**（如引用「跳過 SA 前置審查」字樣、或討論 hook 本身）仍誤觸——keyword-pair 分不清「意圖跳過」與「引用/討論」（lexical false positive / over-broad match）。

### 改動（`hooks/process-skip-guard-hook.py`）

新增**保守的引用/討論抑制**（建議套用於 `detect_skip_intent` 或 SKIP_SA_REVIEW 命中後的後置判定）：
1. **引用 span 抑制**：命中關鍵字落在引用行（行首 `>` 或 `▎` 等 quote 前綴）或明顯的 echo/system-reminder 區段內 → 視為引用非意圖。
2. **meta 討論詞共現抑制**：輸入同時含強 meta 討論詞（如「假陽性 / 誤觸 / 固化 / 邊界 / hook 提示 / false positive / spurious」）時，判為討論 hook 本身而非執行跳過 → 靜音。

**方向強制保守**：抑制只在「明確是引用/討論」時生效。寧可偶爾誤觸（PM 依 skip-gate.md「SA 前置審查假陽性邊界」judge-past）**也絕不可漏放真正的跳過意圖**——false negative 會使防護失效，比 false positive 危險得多。設計時若不確定，傾向不抑制（維持觸發）。

### 測試要求（`hooks/tests/test_process_skip_guard_hook.py`）

- 既有測試全綠（含 type/phase guard 靜音）。
- 新增抑制測試：引用 span（`> 跳過 SA...`）、meta 詞共現（「這個跳過 SA 的誤觸是假陽性」）→ 不觸發 SKIP_SA_REVIEW。
- 新增 **false-negative 防回歸**測試：真正跳過意圖（「這次直接跳過 SA 審查不派 saffron」無引用/meta 語境）**仍觸發**。
- 跑測試：`uv run --project hooks pytest hooks/tests/test_process_skip_guard_hook.py -v`；N>=1 全綠。

### 相關 Error Pattern

文件側對稱：skip-gate.md「SA 前置審查提醒的假陽性邊界」（W1-004）；模式類比 cd 的 IMP-008（文件）+ bash-edit-guard DENY（hook，PR #6）。
