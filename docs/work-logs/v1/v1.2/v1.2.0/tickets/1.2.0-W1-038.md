## Solution

### 落地內容（`.claude/scripts/sync-claude-push.py` 補撞號對稱模組）

鏡像 `sync-claude-pull.py` 行 1329+ 的撞號邏輯（`parse_pc_filename` /
`_next_available_pc_number` / lineage 註記格式），新增 push 端對稱函式：

| 函式 | 職責 |
|------|------|
| `parse_pc_filename` | 解析 bare PC-NNN（zero-padded PC-010 視為 10；前綴 PC-V1-/APP-/TUNL- 排除） |
| `parse_local_lineage` | 解析本地 lineage 註記 →（上游號, 本地號），規則 1 辨識依據 |
| `build_upstream_pc_index` | 掃 clone 上游 error-patterns 建 {號: set(slug)} + by_slug；容納 mess（同號多 slug） |
| `_next_available_canonical_number` | 鏡像 pull，加 `reserved` 集合避免同輪多檔撞同一新號 |
| `plan_push_pc_action` | per-file 決定 none / exclude_lineage_artifact / assign_canonical（純函式） |
| `plan_upstream_mess_reconciliation` | 規則 3 對帳：辨識同 slug 重複號（誤推鏡像）；不誤報合法同號多 pattern |
| `apply_push_pc_collision` | orchestrator：對 temp_dir 套規則 1+2（dry_run 旗標） |
| `writeback_local_canonical_assignments` | push 成功後把賦 canonical 結果回寫本地 working tree |

### 三規則對映

1. **辨識 pull 重編 artifact 不外推**：本地含 lineage 標記 → `exclude_lineage_artifact`，從 temp_dir unlink（既存上游重複份隨 overlay+unlink 由 git add -A 帶刪除）。
2. **本地原生撞號賦 canonical**：本地 bare slug 不在上游此號集合且該號被別 slug 佔 → `_next_available_canonical_number` 賦新號 + 注入 lineage；push 成功後回寫本地。
3. **首跑對帳**：列出上游同 slug 重複號（精準訊號），dry-run 印計畫。

### 整合點（main）

- clone 後、copy 覆蓋前 snapshot `upstream_pc_index`（必須在 copy 前取上游純態）+ 印對帳計畫。
- `copy_filtered_from_staging` 後、git add 前呼叫 `apply_push_pc_collision`。
- push 成功後 `writeback_local_canonical_assignments`（規則 1.5：記憶寫入晚於確認世界副作用）。
- `--dry-run` 經 `_dry_run_report_pc_collision` 純讀印對帳 + 本地處置模擬。

### dry-run 驗證輸出（首跑辨識 6 檔 mess）

```
上游既存 mess（同 slug 重複號）：2 項
  [重複] slug=ui-test-...-unreachable canonical=PC-178 / 誤推鏡像=PC-182（高號該移除）
  [重複] slug=defensive-rule-...-overapplication canonical=PC-177 / 誤推鏡像=PC-181（高號該移除）
本地 PC 處置模擬：排除 4（PC-171/181/182/183 lineage artifact）/ 賦 canonical 0 / 原樣推送 177
```

修正前一版誤報 PC-010/019/020/030/105 為撞號（zero-padded + 同號合法多 pattern
false positive），修正後對合法配號零誤動。

### 已知限制（記錄，非本 ticket 解）

先前不對稱 push 已把本地原生 PC-177-malformed / PC-178-ticket-body 推上框架
canonical 號，上游此號變「foreign canonical + native intruder」兩 slug。單看上游
無法判別哪個是 native intruder（需本地全索引對照），故不自動讓位這類已誤推 native，
避免誤刪框架 canonical。可操作的去重訊號（181/182）已精準處理。剩餘兩 slug 並存
待後續以本地索引對照解（規則 5：發現即建 ticket，見收尾）。

### 紀律
