## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

terminal_screen_test.dart 缺少 W2 修復功能的 widget test 覆蓋：back 按鈕導航、text input submit、connected 狀態操作。W3-004 已補 disconnect 按鈕和 disconnected back 測試，但 text input submit 和 connecting back 導航尚無覆蓋。

### Context Bundle

現有測試（terminal_screen_test.dart 約 300 行）：連線成功畫面、resize、disconnect 按鈕、disconnected back、error 畫面、列舉完整性。
缺：text input submit 行為（_submitInput 呼叫 protocol.encodeInput + sendData）、connecting/idle back 按鈕點擊後 go_router 導航回 /、connected 狀態 status bar 呈現。
注意：W3-002 已將 developer.log 遷移為 AppLogger，新測試應用 AppLogger 而非 developer.log。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

在 `terminal_screen_test.dart` 新增 3 個 widget test：

1. **back 按鈕導航**：使用 GoRouter 包裝測試 app，斷線後點擊 Back 按鈕驗證導航至 `/`。
2. **text input submit**：connected 狀態下於 TextField 輸入文字後 submit（TextInputAction.send），驗證 `protocol.encodeInput('hello\n')` 編碼結果送到 WS sink。
3. **connected 狀態 status bar**：驗證 connected 狀態下 status bar 顯示 `Terminal` 標題和 `Connected` 文字。
