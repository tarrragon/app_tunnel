## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 設計

三層拆分（通用引擎 / language profile / dart 綁定 shim）：

1. **presence-detection-hook.py（通用引擎）**：只懂流程（三類偵測 + override marker
   + 只掃變更內容），無任何語言假設。依副檔名選 profile；**無對應 profile → no-op
   (exit 0)**，是安全上游的核心保證。hook_utils 統一日誌、main 返回 int、logger 在
   main 內初始化。
2. **config/presence_profiles.py（profile registry）**：PresenceProfile dataclass
   = {extensions, skip_patterns, override_markers, string/color/magic 的 detect+exclude}。
   dart profile 1:1 移植原 pattern；python profile 為 stub 證可擴充（無顏色概念 →
   color_detect 留空，引擎自動跳過該類）。新增語言只需新增 profile，引擎不動。
3. **dart-presence-detection-hook.py（shim）**：綁 dart profile，對外維持原單參數介面
   （detect_violations(content)/should_skip_file(path)/...），保 34 既有測試契約。

settings.json Edit/Write 兩處註冊改指通用引擎（引擎依副檔名 .dart 自動取得 dart
profile，runtime 等價）。

### 上游安全性

非 Flutter 專案 pull profile 集合後，對其 .js/.ts/.go 等檔案因無對應 profile 而
no-op，.dart 偵測規則不洩漏，可安全散佈上游。
