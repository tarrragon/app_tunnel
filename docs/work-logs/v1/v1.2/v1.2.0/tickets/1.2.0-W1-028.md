### 補：第二段缺口 — cwd 污染致 ticket 寫錯位置

worktree agent 派發後，PM 持久 shell cwd 被留在最後一個 worktree 目錄（agent-aa139b...）內。後續裸 `ticket create` 以 cwd 的 git toplevel 為基準，將 1.2.0-W1-028.md 寫入該 worktree 而非 main，需手動 cp 恢復。

世界平面證據：`pwd` 回 `.claude/worktrees/agent-aa139b000c8243a31`，028.md 生成於該 worktree 的 docs/。

防護方向：(a) worktree 相關 ticket 寫入命令一律 `(cd <project-root> && ticket ...)` 子 shell 包裹；(b) 評估 worktree 派發後自動重置 cwd 的 hook；(c) ticket CLI 偵測 cwd 在 worktree 內時警告或改寫 main toplevel。併入本 ANA 的 worktree 工作流守護設計。

## 重現實驗結果
### 兩次事故世界平面證據（本 session 實發）

**事故一：未提交產品碼遺失（018）**
- worktree 分支 log 僅含 ticket-md append-log commit，無 .dart code commit（`git log --stat <branch>` 證實）。
- parsley 建檔+測試通過但未 `git add/commit` 產品碼；`complete` 僅 auto-stage metadata 未 commit。
- PM merge 僅得 ticket md（merge stat「1 file changed」紅旗未察）→ `worktree remove --force` 刪除未提交工作樹 → `git fsck` 無 unreachable 含該檔 → 永久遺失。

**事故二：cwd 污染 → main repo 分支漂移（024/025）**
- `pwd` 回 worktree 路徑（非 main repo）；後續 ticket CLI 以 cwd git toplevel 為基準寫錯位置。
- main repo HEAD 漂移到 `feat/1.2.0-W1-025-terminal-frame`（`git branch --show-current` 證實，疑 agent/hook 在 main repo 建支）。
- PM 的 `git -C <main> merge 024` 落在誤切分支、非 main；`push` 顯示 up-to-date、`git log main` 無 W1-024 → 揭穿。
- 我一度用 cwd-relative grep「確認內容在」推翻正確的 `git show main:`=0 → 違反 tool-output-trust 規則 3（固定值才可信）。

兩事故碼均無淨損（worktree 分支保全），但揭露工作流結構缺口。

## Solution
<!-- Schema[ANA/Solution]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 守護設計（兩事故對應兩守護）

**Guard A — 交付物提交驗證（防事故一）**
- 規則層：worktree 完成協議明訂「parsley/實作 agent 完成前必須 git add+commit 產品碼進 worktree 分支；ticket CLI 只 commit metadata」（已在本 session 重派 prompt 驗證有效，固化進 agent 完成協議 + worktree-operations）。
- PM 層硬規則：merge 後、worktree remove 前，必須 `git show main:<where.files>` 或 `git ls-tree -r main` 驗證 ticket 交付物確在 main，才可 remove（本 session 內化、6+ 次零重演驗證有效）。
- Hook 層（可自動化部分）：worktree remove 前若該 worktree 分支有未 merge 進 main 的 where.files 變更 → 警告/阻擋。

**Guard B — 分支漂移 / cwd 污染偵測（防事故二）**
- 規則層：worktree 相關 ticket 寫入命令一律 `(cd <project-root> && ticket ...)` 子 shell 包裹；PM 每次 worktree 派發後 `pwd` + `git -C <main> branch --show-current` 確認未漂移（本 session 內化）。
- 信任層：固化 tool-output-trust「cwd-relative 查詢在污染下會騙人，關鍵事實一律用 `git show <ref>:` 等固定值」（我違反一次的教訓）。
- Hook 層：pre-worktree-dispatch 檢查 main repo HEAD 是否仍在預期分支（偵測 agent 在 main repo 誤建支）。
