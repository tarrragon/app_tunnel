## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

Flutter 端無統一 logging 層。12 處 developer.log 散落在 connection_manager.dart（7 處）和 terminal_screen.dart（5 處），name 參數不一致，無法統一開關或格式化。BiometricService 和 CredentialRepository 無任何 log。

### 影響範圍

新建：app/lib/core/logging/app_logger.dart
修改：connection_manager.dart、terminal_screen.dart 的 developer.log 改為 AppLogger
補 log：biometric_service.dart、credential_repository.dart、ttyd_protocol.dart

### Context Bundle

現有 developer.log 分佈：connection_manager.dart 7 處、terminal_screen.dart 5 處。
觀測性規則要求：統一日誌工具（rules/core/observability-rules.md 規則 2）、關鍵流程三階段 log（規則 4）。
AppLogger 設計：封裝 developer.log，提供 info/warning/error 分級，統一 name 格式 [Component]。
遷移策略：先建 AppLogger -> 逐檔替換 developer.log -> 補 BiometricService/CredentialRepository/TtydProtocol log 點。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

### AppLogger wrapper

建立 `app/lib/core/logging/app_logger.dart`，封裝 `developer.log`，提供三個靜態方法：
- `info()` — 一般資訊（連線步驟、狀態轉換、操作結果）
- `warning()` — 警告（非致命異常、降級處理），訊息前綴 `[WARNING]`
- `error()` — 錯誤（操作失敗、連線中斷），訊息前綴 `[ERROR]`

所有方法接受 `component` 參數作為 `developer.log` 的 `name`，統一格式。

### 遷移 12 處散落 developer.log

| 檔案 | 原 developer.log 數 | 遷移後 |
|------|---------------------|--------|
| connection_manager.dart | 7 | 7 AppLogger 呼叫（info x5, error x2） |
| terminal_screen.dart | 5 | 5 AppLogger.info 呼叫 |

移除 `dart:developer` import，改用 `package:app_tunnel/core/logging/app_logger.dart`。
移除未使用的 `dart:io` import（dart analyze 清理）。

### 補 log 點

| 元件 | 新增 log 點 |
|------|-----------|
| TtydProtocol | encodeInput, encodeResize, decodeOutput, buildHeaders, buildAuthTokenFrame（5 處 info） |
| LocalAuthBiometricService | isAvailable, authenticate 啟動/結果/不支援（4 處 info + 1 處 warning） |
| SecureStorageCredentialRepository | save, load, delete, exists（4 處 info） |
