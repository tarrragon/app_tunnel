## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

套用 ANA 1.2.0-W1-014 的 ANSI 16 色重配常數（已落地於 `app/lib/core/theme/app_colors.dart`）至 `terminal/renderer/`，遵循語意保真鐵則。

| 檔案 | 變更 |
|------|------|
| `ansi_parser.dart` | `standardColors`/`brightColors` 由內聯 xterm 值改引 `AppColors.kAnsiStandard`/`kAnsiBright`。SGR 索引邏輯（`code-30`/`code-90`/`code-40`/`code-100`）零改動。 |
| `terminal_renderer.dart` | `_defaultForegroundColor`（原 0xFFE0E0E0）→ `AppColors.kAnsiDefaultForeground`；`_backgroundColor`（原 0xFF1E1E1E）→ `AppColors.kAnsiDefaultBackground`。 |
| `ansi_recolor_test.dart`（新增） | 驗證 palette 1:1 對應 AppColors、SGR 30-37/90-97 索引→重配值對應、bright[i] 明度 > standard[i]（鐵則 2 明度序）。 |
| `terminal_renderer_test.dart` | 背景斷言由硬編碼 `0xFF1E1E1E` 改為 `AppColors.kAnsiDefaultBackground`。 |

### 鐵則遵循

- 色相身分不動：僅換常數值，索引邏輯與 SGR 語意映射完全保留。
- normal/bright 明度序：測試以 WCAG 相對亮度驗證 bright 恆高於對應 normal。
- fallback 路徑保留：`kAnsiXtermFallback*` 常數仍在 app_colors，未動。
- 紀律：僅動 renderer/，未碰畫面層（terminal_screen 等）。
