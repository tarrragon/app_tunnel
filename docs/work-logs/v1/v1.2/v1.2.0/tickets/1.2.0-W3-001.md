## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

W2 實機測試（W2-001~004）暴露企劃階段三大設計維度缺失：

1. **LOG**：D8 只規劃 server audit log，client-side 連線生命週期/protocol 訊息/認證結果零 log
2. **UX**：§1 操作盤點有失敗情境但未落地為畫面狀態機——每個畫面的狀態/操作/退出路徑未設計
3. **Testing**：D9 未區分測試層級——unit(mock) 全綠但 protocol integration test 不存在，192 測試遮蔽真實行為

### 影響範圍

| 維度 | 影響元件 | 嚴重度 |
|------|---------|-------|
| LOG | ConnectionManager, TerminalScreen, TtydProtocol, BiometricService, CredentialRepository, EnrollmentScreen | P1 |
| UX | TerminalScreen(idle/connected 缺 back/disconnect), 畫面狀態機未設計 | P1 |
| Testing | connection_flow_test(fake integration), protocol integration(不存在), widget test(不覆蓋導航) | P1 |

### 相關 Error Pattern

與 quality-baseline 規則 1 邊界相關：「測試綠燈不等於 Runtime 正確」（PC-165）

### 盤點數據

**LOG 現狀**：
- ConnectionManager: 10 個 developer.log（W2 hotfix 補的）
- TerminalScreen: 5 個 developer.log（W2 hotfix）
- TtydProtocol/BiometricService/CredentialRepository/EnrollmentScreen: 零 log
- Go proxy: slog JSON audit（僅連線事件），缺 WS upgrade 細節
- 無統一 logging 層，全部散落 developer.log

**UX 現狀**：
- HomeScreen: OK（root，有 Connect + Enroll）
- EnrollmentScreen/QrScannerScreen: OK（有 Navigator.pop）
- TerminalScreen idle: 缺 back
- TerminalScreen connected: 缺主動斷線/返回
- TerminalScreen error/disconnected/connecting: W2-001 已修 back

**Testing 現狀**（20 檔 ~192 test cases）：
- Unit(mock): 16 檔 ~184 cases，全用 Fake
- Integration(fake): 3 檔 ~22 cases，FakeWebSocketChannel
- Widget test: 3 檔 ~12 cases，不覆蓋 back/input
- Protocol integration(real): 0 檔 0 cases
- Go server integration_test.go: 存在（app 端不對稱）
