## Solution
<!-- Schema[DOC/Solution]: 免填（DOC 類型以變更摘要取代）（.claude/pm-rules/ticket-body-schema.md） -->

變更摘要：`.claude/pm-rules/skip-gate.md` 豁免段後新增「SA 前置審查提醒的假陽性邊界」子節。

- 三假陽性情境（PM 可 judge-past 但須記 rationale）：任務已完成（pre-implementation gate 結構性 moot）/ 修復形態 + PR review（hook 自承「修復類可能不需要」）/ 僅討論引用 skip-gate（關鍵字偵測誤觸）。
- 必 honor（不可 judge-past）：新功能或架構變更的 pre-implementation 階段——SA 前置審查為實質防護。
- 含 Why/Consequence/Action：關鍵字偵測無法區分「意圖跳過」與「結構不適用/僅討論」，缺邊界致每次臨場判斷或誤 honor moot 審查。

動機：process-skip-guard hook 本 session 多次對「已完成任務」與「引用 skip-gate 字樣的訊息」誤觸。為框架資產（pm-rule），待 sync-push 上游。
