### 兩守護落地（規則層 + hook 層 + 信任層）

**Guard A — 交付物提交驗證（防 028 事故一：未提交碼遭 force-remove 遺失）**
- hook 層：`worktree-remove-deliverable-check-hook.py`（PreToolUse:Bash）。偵測 `git worktree remove`，解析目標路徑 → 查對應分支 → `git log main..<branch>` 有觸及檔案的未合併 commit 則阻擋（exit 2）。`--force` 不繞過（hook 先於 git）。批量 while-read 腳本無法靜態解析路徑時放行（不誤擋）。
- 規則層：worktree-operations 階段 3 補「Guard A：remove 前用 `git show main:<file>` / `git ls-tree -r main` 固定值驗證交付物在 main」PM 硬規則；Subagent isolation 章節補「實作 agent commit 紀律」。
- agent 層：AGENT_PRELOAD git 操作規則新增「worktree 隔離派發時必須 commit 產品碼進 worktree 分支」（凌駕 Phase 3b+ 禁 commit，因隔離分支 commit 是交付物可見性前提，非主 repo 提交，未違反 PC-024 push 禁令）。

**Guard B — 分支漂移偵測（防 028 事故二：main repo 漂到 feat 致 merge 落錯處）**
- hook 層：`worktree-pre-dispatch-branch-drift-hook.py`（PreToolUse:Agent）。`isolation: worktree` 派發時查主 repo `git symbolic-ref --short HEAD`，非 main/master 則阻擋（exit 2）。detached/git 失敗放行（已記錄日誌）。
- 規則層：worktree-operations 階段 1 補「Guard B 前置：派發前 `git -C <root> branch --show-current` 確認未漂移」+ 禁 cwd-relative 推斷。
- 信任層：規則文字明示「關鍵事實用固定值/明示 ref，勿用 cwd-relative 查詢」（引 tool-output-trust 規則 3）。

**settings.json 註冊**：Guard A 入 PreToolUse:Bash（timeout 15000）；Guard B 入 PreToolUse:Agent（timeout 10000）。
