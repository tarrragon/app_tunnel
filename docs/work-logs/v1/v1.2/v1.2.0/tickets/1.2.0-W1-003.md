### 工作環境（重要）

### 問題根因

`bash-edit-guard-hook.py` 的 bare cd 偵測已完整（正確排除子 shell / git -C / uv -d / 專案根還原），但行為是 `permission_decision="allow"` + warning。warning-only 已三度被忽略（IMP-008），cwd 汙染導致下游工具 `get_project_root()` 誤判、連環假性除錯。依設計（子 shell / git -C / uv -d / 絕對路徑）從不需持久 cd → bare cd 永遠是錯誤 → 應在執行前 DENY。

### 改動（檔案：`hooks/bash-edit-guard-hook.py`）

1. **bare cd 分支 allow → deny**：`main()` 中 `_find_bare_cd_target` 命中時，`emit_hook_output` 改 `permission_decision="deny"`，reason 含命中 target + 指引（改用 `git -C <abs>` / 子 shell `(cd ... && cmd)` / `uv -d <dir>`）。**sed/perl 原地編輯維持 warn（allow）不變**——僅 cd 分支改 deny；若同時命中 cd + edit，以 deny 為準。
2. **heredoc/quoted FP 抑制**（DENY 前必補，避免誤擋合法 script 撰寫）：`_find_bare_cd_target` 命中點若落在 heredoc body（`<<EOF ... EOF`）或單/雙引號字串內，視為字面 cd 不算 bare cd。實作建議：偵測 heredoc 區段範圍 + quote 狀態，命中點 offset 落在這些區段內則 skip（比照既有子 shell 括號深度的 offset 判定）。

### 測試要求（`hooks/tests/test_bash_edit_guard_hook.py`）

- 既有測試全綠（子 shell 排除 / git -C / 專案根還原 / 多行 / sed-perl warn）。
- 新增 deny 行為測試：bare cd（行首 / `&& cd` / `; cd` / `|| cd` / pushd）→ permission_decision == "deny"。
- 新增 FP 抑制測試：heredoc body 內 `cd /x`、單引號 `'cd /x'`、雙引號 `"cd /x"` → 不 deny（allow）。
- sed/perl 原地編輯仍 == warn（allow），未被 deny 波及。
- 跑測試：`uv run --project hooks pytest hooks/tests/test_bash_edit_guard_hook.py -v`；N>=1 全綠。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->

**實作於** clone `/tmp/tcr-pr` 分支 `feat/bash-edit-guard-deny-bare-cd`，commit `84d5b33`。

### 改動 1：bare cd 分支 allow → deny（`hooks/bash-edit-guard-hook.py`）

- `main()` 拆分輸出路徑：`bare_cd_target is not None` → `permission_decision="deny"`；僅 sed/perl 命中 → 維持 `allow`。cd + edit 同時命中走 deny 分支（deny 為準）。
- 新增 `_bare_cd_deny_reason(target)`：reason 含命中 target + 三種替代指引（`git -C <abs>` / 子 shell `(cd <dir> && <cmd>)` / `uv -d <dir>`）+ 規則路徑。
- 模組 docstring「行為」段更新為模式 A（warn）/ 模式 B（deny）雙路徑描述。

### 改動 2：heredoc / quoted FP 抑制（DENY 配套）

- 新增 `_literal_offset_ranges(command)`：計算字面區段 offset：
  - heredoc body：標頭 `<<` / `<<-` + 可選 quoted delim（`<<'EOF'`），body 從標頭行換行後起到結束 delimiter 行（行首可選空白 + delimiter 單獨成行）；無結束 delim 視為延伸至命令尾。
  - 單/雙引號字串：逐字尋找成對引號（含引號本身）。
- 新增 `_offset_in_literal(offset, ranges)`。
- `_find_bare_cd_target` 迴圈內新增「排除 0」：命中 `cd/pushd` 關鍵字起點落在字面區段則 `continue`。
- 抑制方向保守（少抑制 → 維持 DENY），不漏放真正裸 cd。
