## Problem Analysis

### 問題根因

VIBE CODING 慣性：app/lib 24 dart 檔 + server/ 4 go 檔中，user-facing 文字全部硬編碼字串、UI 尺寸/間距/timeout 為魔術數字、顏色為散落的 Color()/Colors. 呼叫。app 無 l10n/ARB 設施、core/constants/ 目錄為空。

### 影響範圍

- **app/lib/**：24 dart 檔，其中 6 檔含 user-facing 硬編碼英文字串、5 檔含魔術數字、4 檔含散落顏色值
- **server/**：3 go 檔含硬編碼中文 user-facing/log 字串、flag 預設值含魔術數字

### 相關 Error Pattern

無直接對應 PC/IMP。屬品質基線（quality-common 1.3 常數管理、1.4 DRY）與 i18n 管理方法論範疇。

## 重現實驗結果

全量 grep 掃描 app/lib/ (24 dart) + server/ (go)，分三類：
1. 中文字元：grep -rnP [\x{4e00}-\x{9fff}]（區分程式碼內嵌 vs 註解）

### 二、i18n 設施方案（從零建 ARB）

### 2.1 ARB 結構

```
app/
  l10n/
    app_en.arb          # 英文（唯一語言，SOT）
  lib/
    l10n/               # flutter gen-l10n 產出（gitignore）
  l10n.yaml             # gen-l10n 設定
  pubspec.yaml          # 新增 flutter_localizations + intl 依賴
```

### 2.2 l10n.yaml 設定

```yaml
arb-dir: l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
output-dir: lib/l10n
synthetic-package: false
nullable-getter: false
```

### 2.3 key 命名規範

三段式：`{feature}.{screen}.{element}`，駝峰命名。

| 規則 | 範例 |
|------|------|
| 功能模組.畫面.元素 | `enrollment.scan.title` → `enrollmentScanTitle` |
| 共用字串用 common 前綴 | `common.cancel` → `commonCancel` |
| 錯誤訊息用 error 中綴 | `terminal.error.auth` → `terminalErrorAuth` |

ARB key 為 camelCase（Flutter gen-l10n 預設）。

### 2.4 pubspec.yaml 變更

```yaml
dependencies:
  flutter_localizations:
    sdk: flutter
  intl: any    # gen-l10n 需要

flutter:
  generate: true
```

### 2.5 MaterialApp 接線

```dart
import 'package:app_tunnel/l10n/app_localizations.dart';

MaterialApp(
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: AppLocalizations.supportedLocales,
  // ...
)
```

使用端：`AppLocalizations.of(context).enrollmentScanTitle`
