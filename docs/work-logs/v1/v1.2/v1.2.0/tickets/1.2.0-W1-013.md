## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因

VIBE CODING 慣性：app/lib 24 dart 檔 + server/ 4 go 檔中，user-facing 文字全部硬編碼字串、UI 尺寸/間距/timeout 為魔術數字、顏色為散落的 Color()/Colors. 呼叫。app 無 l10n/ARB 設施、core/constants/ 目錄為空。

### 影響範圍

- **app/lib/**：24 dart 檔，其中 6 檔含 user-facing 硬編碼英文字串、5 檔含魔術數字、4 檔含散落顏色值
- **server/**：3 go 檔含硬編碼中文 user-facing/log 字串、flag 預設值含魔術數字

### 相關 Error Pattern

無直接對應 PC/IMP。屬品質基線（quality-common 1.3 常數管理、1.4 DRY）與 i18n 管理方法論範疇。

## 重現實驗結果
### 實驗方法

全量 grep 掃描 app/lib/ (24 dart) + server/ (go)，分三類：
1. 中文字元：grep -rnP [\x{4e00}-\x{9fff}]（區分程式碼內嵌 vs 註解）
2. 顏色：grep -rnP (Color\(|Colors\.)
3. 魔術數字：grep 含 width/height/padding/fontSize/timeout/Duration/port/size/radius 的數字字面量（排除註解、hex byte、case label）
4. User-facing 英文字串：grep Text()/title:/label:/SnackBar/AlertDialog 取字串字面量

### 實驗執行

四組 grep 均已完成，raw 結果見下方分類盤點。

### 實驗發現

**已驗證事實**：
- app/lib/ 中文字元：100% 位於 // 註解內（doc comments），無 user-facing 中文字串。所有 user-facing 文字為英文硬編碼。
- server/ 中文字元：分兩類——(a) 註解中文 (b) fmt/log 字串中文（user-facing + 稽核 log）。server 是 CLI 工具，user-facing 中文在 enroll.go + main.go + secret.go。
- app 無任何 l10n 設施（無 l10n/ 目錄、pubspec.yaml 無 flutter_localizations/intl 依賴、無 ARB 檔）。
- core/constants/ 目錄存在但為空。

**仍未驗證假設**：
- 顏色值是否與 ANA-B(1.2.0-W1-014) 設計 token 衝突——待 014 產出後對照。
